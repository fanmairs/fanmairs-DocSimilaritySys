# 文档相似度比对与抄袭检测工具 (Document Similarity System)

## 1. 项目简介
本项目是一个基于 Python 实现的轻量级、可解释的文档相似度比对与抄袭检测工具。主要用于解决学术论文、技术报告等电子文档的版权保护问题。与主流的“黑盒”商业查重软件或高算力需求的深度学习模型不同，本项目回归数学基础，采用 TF-IDF 和 LSA（潜在语义分析）等传统且高效的 NLP 算法，能够有效识别“直接抄袭”和同义词替换的“洗稿”行为。

## 2. 核心技术栈
*   **编程语言**: Python 3.x
*   **中文分词**: Jieba
*   **科学计算**: NumPy
*   **核心算法**: 
    *   **TF-IDF** (词频-逆文档频率): 提取文本字面特征，构建高维稀疏矩阵。
    *   **LSA / SVD** (潜在语义分析 / 奇异值分解): 对高维特征进行降维，在低维空间捕捉文本的深层语义逻辑，解决“同义词”漏检问题。
    *   **Cosine Similarity** (余弦相似度): 量化文档间的向量夹角差异，得出最终查重率。

## 3. 项目目录结构

```text
DocSimilaritySys/
│
├── data/                  # 测试文档与对比语料库目录
│   ├── 检测文档.txt       # 默认的待检测文档
│   ├── doc2.txt           # 默认的参考文档之一 (用于测试洗稿)
│   └── doc3.txt           # 其他测试文档 (如："讨厌红楼梦")
│
├── dicts/                 # 词典资源目录
│   └── stopwords.txt      # 停用词表 (用于过滤“的”、“是”等无意义词汇)
│
├── preprocess.py          # 核心模块 1: 文本预处理 (清洗、分词、去停用词)
├── vectorize.py           # 核心模块 2: 特征向量化 (白盒化实现的 TF-IDF)
├── lsa_svd.py             # 核心模块 3: 潜在语义分析 (利用 SVD 进行矩阵降维)
├── similarity.py          # 核心模块 4: 相似度计算 (余弦相似度及基础测试逻辑)
├── detector.py            # 扩展模块: 基于滑动窗口的细粒度段落级查重 (进阶功能)
│
├── main.py                # 系统主入口 (整合所有模块，提供命令行调用与报告输出)
└── 开题报告.txt           # 项目原始需求说明
```

## 4. 如何使用

本系统支持灵活的命令行调用，可以实现单文档比对或批量目录查重。

### 环境准备
确保已安装所需的第三方库：
```bash
pip install jieba numpy
```

### 运行方式

**方式 1：默认运行 (最简单)**
在 PyCharm 中直接点击 `main.py` 的运行按钮，或者在终端输入：
```bash
python main.py
```
> **行为说明**：系统会自动将 `data/检测文档.txt` 作为目标文档，并与 `data/` 目录下的所有其他 `.txt` 文件（如 doc2, doc3）进行比对，最终输出相似度排名报告。

**方式 2：指定目标文档，与默认 data 目录比对**
```bash
python main.py "您的文档路径.txt"
# 示例：
python main.py "开题报告.txt"
```

**方式 3：指定目标文档和特定的参考库目录 (完整自定义)**
```bash
python main.py "待检测文档.txt" "参考文档文件夹/"
# 示例：
python main.py data/检测文档.txt data/
```

### 报告解读
运行后，系统会在终端输出一份格式化的报告：
*   **相似度(LSA)**: 代表深度语义层面的相似度，数值高说明存在“洗稿”或逻辑抄袭。
*   **相似度(TF-IDF)**: 代表字面完全匹配的相似度。
*   **警示标记**: 
    *   🚨: 相似度 > 70%，极度疑似抄袭。
    *   ⚠️: 相似度 > 40%，疑似洗稿，需人工复核。

## 5. 设计亮点 (答辩要点)
1.  **完全白盒化**: 所有的 TF-IDF 计算、SVD 截断降维过程均由基础数学库 (NumPy) 手写实现，未使用 `scikit-learn` 等高度封装的库，算法流程透明，便于理解和解释。
2.  **抗洗稿能力 (动态 LSA 降维)**: 引入了简单的背景语料库和 LSA 降维打击，使得系统不仅能发现一模一样的抄袭，还能识别出“互联网”替换为“网络科技”这种同义词洗稿行为。并且实现了**自适应动态维度截断**，解决了小样本下余弦相似度过度平滑导致误判的数学难题。
3.  **精准的“雷神之锤”定位 (N-gram 自然句滑动窗口)**: 
    * 摒弃了粗糙的“按字数物理切分”，采用基于中文标点符号的**正则表达式自然句切分**。
    * 首创轻量级的**滑动重叠句群（N-gram Sentences）**技术。通过将相邻的句子拼接对比，不仅解决了单句特征过少的问题，更能精准对抗“拆分/合并长短句”的高级洗稿手段。当整体相似度超标时，系统会自动打印出具体的“疑似抄袭片段”对比供人工复核。
4.  **工程实用性**: 优化了文件读取逻辑，自动兼容 `UTF-8` 和 `GBK` 编码，避免了中文系统常见的乱码报错；支持自动扫描目录并输出 Top N 排名；内置了扩充版的高精度中文停用词表。
